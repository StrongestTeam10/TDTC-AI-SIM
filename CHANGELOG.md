# Changelog

## [0.1.0] - 2026-07-21
### 추가 - 초기 구축
- FastAPI + Mesa 3.5.1 기반 시뮬레이션 엔진 스캐폴딩
- 파이프라인 A(`POST /simulate/snapshot`): 센서 실측 기반 오브젝트 배치 + 위험도 산출
- 파이프라인 B(`POST /simulate/scenario`): 초기 배치까지만 구현, 이벤트 모델은 후속 과제
- 헬스체크 `/health`, `/health/db`
- 공간 처리(space.py): GeoJSON 파싱, 위경도 <-> 로컬 미터 좌표 변환, 골목형 유효 통로폭 산출
- 배치 로직(placement.py): 균등 / 중심선 가중 분포 2가지 전략
- 위험도(risk.py): 행안부·NFPA·Fruin·Still 기준 근거, 4개 지표 종합
- DB 계층: Supabase 커넥션 풀 + 조회 리포지토리, 위험도 결과 mrkrisk01m 기록

### 설계 판단 기록
- 위험도 가중치 재정규화 적용: 결측 지표의 가중치를 제외하고 재배분.
  미적용 시 밀집도가 임계를 넘어도 종합 점수가 최대 65점에 그쳐 critical 등급 도달이 불가능했음.
- 병목 지표를 '인원/통로폭'에서 '대피 소요 시간'으로 재정의.
  전자는 구역 길이를 무시해 300명 수준에서 이미 포화되어 변별력이 없었음.
- 밀집도 단독 안전 오버라이드 추가: 압사는 밀집도만으로 발생하므로 종합 평균에 희석되지 않도록 함.
- DB connect_timeout=5초 명시: psycopg2 기본값은 무한 대기라, DB 도달 불가 시 앱이 영구히 기동되지 않음.
- lifespan에서 DB 초기화 실패를 치명적 오류로 처리하지 않음: DB 일시 장애와 앱 기동을 분리.
